CESM CASEROOT env_build.xml file

Model Version: CESM2.0
HTML created on: 2017-06-21

Description

The case.build script reads the XML settings in the env_build.xml file in order to
determine the machine specific compile and build commands used for a specific case
and machine.

Interface Tools

CESM2.0 supports multiple build groups. The xmlquery and xmlchange script tools
accept the --subgroup command line argument to define one of the build groups. Changes to the build settings can be made prior to calling case.build for the first time..
Subsequent changes may require case.build --clean or case.build --clean-all followed by case.build.

Variable names delimited by 

{{ }}
are substituted at run-time and should not be modified.

Build Group Settings <group id="build_component_cism"> :

Entry ID/Value Definitions <entry> :
  • Attributes :
    • id = CISM_USE_TRILINOS
    • value = FALSE
  • Data Type <type> : logical
  • Valid Values <valid_values> : TRUE,FALSE
  • Description <desc> : TRUE implies CISM is built with support for the trilinos library When using the higher-order dynamical cores in CISM, one option is to use the trilinos solver library for sparse matrix solves. In order to exercise that option, you need to build CISM with trilinos support, and link to the trilinos libraries when building the CESM executable. Setting this variable to TRUE accomplishes these things. Note that you can set this to TRUE and still choose not to use the trilinos solvers for the sparse matrix solves. Also, note that trilinos is only available on a limited set of machines; this must remain FALSE to run CISM on a machine that does not have trilinos, or for which the CESM scripts are not aware of the existence of trilinos.
